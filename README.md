Битовые карты в ядре Linux (ex_bitmap)

Проект демонстрирует работу с битовыми картами (bitmaps) в пространстве ядра.
Описание

Модуль ядра ex_bitmap.ko реализует простую систему управления памятью, где каждому биту в битовой карте соответствует блок памяти. Модуль демонстрирует базовые операции:

    Инициализация карты (все блоки свободны).
    Выделение блоков (установка битов).
    Поиск первого свободного блока.
    Освобождение блоков (сброс битов).

Все ключевые действия логируются в системный журнал (dmesg).
Структура проекта

    ex_bitmap.c — исходный код модуля ядра.
    Makefile — файл для сборки модуля.

Сборка и запуск

Для сборки и проверки модуля выполните следующие команды в терминале из директории проекта.
1. Сборка модуля

make

Эта команда скомпилирует исходный код и создаст файл ex_bitmap.ko.
2. Загрузка модуля в ядро

sudo insmod ex_bitmap.ko

После выполнения этой команды проверьте вывод системного журнала, чтобы увидеть лог работы модуля:

dmesg | tail

Вы должны увидеть сообщения о выделении и освобождении блоков памяти.
3. Выгрузка модуля

sudo rmmod ex_bitmap

Снова проверьте лог:
dmesg | tail

Вы увидите сообщение о завершении работы модуля.
Описание Makefile

В репозитории используется стандартный Makefile для сборки модулей ядра. Он поддерживает следующие цели:

    make (или make all) — сборка модуля.
    make clean — очистка файлов сборки (.o, .ko, .mod.* и др.).
    make format — форматирование исходного кода с помощью clang-format.
    make check — автоматическая проверка: сборка, загрузка, просмотр лога, выгрузка и повторный просмотр лога.
